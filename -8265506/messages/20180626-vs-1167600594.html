  • 內容：
    • 依賀眾提市場需求，排出優先順序，再來開發
    • 中華電信：公司端需求
      • 平台統一管控
      • 節能省電功能
      • 異常自動通報（需精確到可判斷出何種異常）
      • 水質狀況
        • 進水廠的TDS vs. 飲水機的TDS => 讓使用者安心
    • 賀眾需求:
      • Step1
        • 遠端集中控制
          • 如臨時颱風假 => 可以遠端設定節電
        • 明確看到濾心壽命還有多久、水質顯示、用電量、用水量、溫度資訊等
        • 智能節電排程
          • 可配合公司 or 學校的行事曆
          • 依不同場域去設置不同節能排程
          • 搭配最佳化殺菌時間
      • Step2
        • 賀眾公司端需求：
          • 省下找承辦人的時間，一個維修人員一個月可以省下3天工時 (一個人一天要跑20個維修件)
          • 串接派工 - 驗收 - 金流
            • 系統送出異常告警(維修碼) => 自動判斷是否要報修 =>要的話一個按鈕自動派工報修
            • 報修完，維修人員拍照（維修前中後）傳到系統
            • 對方承辦人按確認鈕 => 自動完成金流服務 or 自動 key in 後續銷帳資料
          • 維修紀錄：知道哪些地方最常壞掉，就可以知道自己怎麼改善設備
          • 關於 UX：
            • 其他公司給他們的公司端管理介面很爛
            • 控制系統可接上公司內部 ERP 系統更好
          • 其他智能功能
            • 殺菌時段的智慧調整
            • 智慧水溫 (自動調整夏天／冬天應該設幾度)
          • 其他可能應用
            • 面板 => 賣廣告
            • 最適當的熱水缸容量到底是多少? 11公升? 13公升?
        • 台科需求:
          •  一些飲水機的控制功能
            • 賀眾：台科大邊做邊提需求，賀眾會再開放控制權給我們，目標盡快商品化
          • 流量計 
            • 一個大概 $100
            • 目前飲水機基板沒有辦法解讀 (轉幾圈要幾公升)
            • 久了磁性會造成精準度的問題
          • 中華電信 NBIoT modem
            • 7/15 會提供 2 個給台科大
  • 相關問題討論
    • 長遠目標: 賣飲水服務，不是飲水機
      • 以維持服務水準為前提節電，沒有要使人不喝水
      • 加熱煮水就是要那麼多電， 我們所要降的是「操作」上的耗電
      • 提升加熱效率、保溫效果，單機少耗電是賀眾本身的功課與目標
    • 飲水機熱缸容量目前機型是 22-20 公升
      • 20公升熱缸設定半缸，不如直接做一個10公升的飲水機保溫效果更好
      • 但切細不是製造廠樂見的，不可能做太多種不同規格的熱缸
      • 採購者的心態: 不會想買剛剛好，只想買越大越好
      • 從數據去歸納出來，找到不同場域的「大中小」各是幾公升
    • 在節電模式內如果有人打斷取水會再回去節電模式嗎? => 可，第24項
      • 目前 default 是最後一個使用完後 1 小時都沒人使用會恢復節電
    • TDS指標：
      • 總溶解固體 > 適飲性指標
      • WHO標準：500以下
      • 每個地區的TDS不一樣 > 取決到喝水者覺得澀或硬
    • 關於進水控制：
      • 進水唯一的停止控制：進水太快、溫度太低，會暫時不進水
      • 目前無法控制進水
    • 何時會更換濾心：
      • 時間到
      • 流量達到 3000 公升
    • 關於賀眾 PCB 板
      • 如果用 GW 2000-4000 去代替裡面的 PCB 可以接受，打算以加價購的方式賣給消費者
      • 用中華電信 NBIoT modem + RS485 + (鉤表) + (Security) 取代原本的板子
    • 台科這個場域基地台有support NB-IoT，目前有兩個門號可使用
