實驗設計 初初版

優先做的五條體驗旅程：
SYS: Zenfone 購前研究 / 欲選購高CP值筆電
OP: 專家買高規格主機板＋自行安裝
MAINTAIN: 新買電腦出現藍屏 
B2C INTERACTION: 幫商品註冊並瞭解保固

體驗旅程相關的研究議題:
O2O support service
每條路徑最理想的情況、目標
competitor analysis, SWOT
新客戶哪個時間點會想成為會員 / 已是華碩會員者 / 曾經送修華碩商品者

實驗流程：
  1. 先透過問卷篩選受試者背景，電子產品的購買、註冊、送修等階段的使用經驗，已是華碩使用者的則詢問目前的使用體驗，目前使用的產品....等。
  2. 訪談使用者，搭配問卷篩選結果建立使用者的 demographic。
  3. 任務一：Zenfone 購前研究。
  4. 任務二：尋覓筆電、購買筆電、到貨電腦故障註冊商品，是要看使用者自發性意願，還是刻意安排？
  5. 任務三：自組電腦者選購高規格主機板＋自行安裝。
  6. 實驗後訪談，請受試者回顧剛剛的體驗歷程。

實驗問卷:
  1. ASQ 以任務“目標”為主，瞭解受試者的主觀感受、意向。
  2. QUIS 來瞭解介面設計的相關內容，mobile、web 介面分開評估。
  3. SUS 用來瞭解華碩整體的線上數位使用體驗。

數據分析：
  1. URL 熱點圖
  2. 哪些節點間的轉化率可以被評估？
  3. 整體跳失率（mobile 的跳失率分別估）
  4. 三個問卷的量化分析

受試者：
  1. 特定年齡層（25~35歲、36~45歲）
  2. 正在用華碩
  3. 已不再用華碩、Win和Android 愛用者（潛在回流購買者）
  4. 不考慮華碩的使用者，譬如：果粉、米粉...。

實驗設備：
  1. 情境卡：引導使用者跨終端地使用。
  2. 遠端錄像程式： morae
  3. 借 eye-tracking 工具（20,000/week）
  4. 主要操作的電腦：備 win 和 MAC
  5. 主要操作的手機：備 android 和 iPhone，可能是受試者個人的裝置。
  6. 紀錄實驗現場：錄音筆、攝影機 X2、相機。
  7. 實驗用產品：向華碩借 Zenfone、故障的整修品筆電、PC...。
  8. 策略聯盟的產品：UXTesting 的 mobile 行為數據監測程式。
