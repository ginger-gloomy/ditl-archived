會議內容:
一、了解DITL對網頁服務的疑惑

　Q:註冊產品、會員制對消費者的好處?
  1. 過去策略是註冊產品才能享有保固，之後演變成不註冊也可享服務。 
  2. 消費者將序號、型號註冊並放入會員庫後，之後使用其他服務可不必重覆填寫。
  3. 軟體更新的話會有推播通知、讓消費者更快知道新資訊。
  4. 有User email的話可隨時收到最新產品及優惠資訊，讓消費者體驗被照顧的感受。
  5. 可享電腦定期健檢。
  6. 網頁上的購物車清單需登入會員，瀏覽紀錄才會被記錄，且可隨時跨裝置查看。
  7. 只要客戶將產品帶入商店，ASUS前線會協助消費者加入會員、註冊服務。
　Q:客戶登入會員對公司方的重要性?
  1. 會員制的未來計畫是以一個人為出發點，蒐集數據整合成大數據以供EC了解消費者族群。
  2. 對未來的產品推出及形象有較明確的目標。
  3. 希望會員提供更詳細的資訊，一方面ASUS可提供生日優惠服務，一方面也可讓內部分析使用者。

　Q:訪談客服人員、瞭解目前支援服務?
  1. "皇家俱樂部-線上更改時間" 路徑，僅為一部分的服務，非主要幹道。

二、其他資訊:
  1. ASUS內部主要分為兩部分，第一部分是筆電、手機、平板，第二部分為顯卡、主機板等內部技術。 (讓技術面的人可再有更深入的瞭解)
  2. ASUS近期看重EC、User flow，若我們能協助，對ASUS會是很大的幫助。
  3. 目前福利品和EC的關聯還沒調整過來，未來會將福利品頁面的flow同步改版。
  4. 目前有新的store版本(尚未上線)，會再提供給我們瞭解。
  5. Mainsite現前最大的目標是依照不同產品的類別進行產品調整，演變為系列版首頁。
  6. support 正在推廣My Asus APP。
  7. 官網是預設消費者在無任何目的的情況下進入網站，希望讓消費者隨意逛官網的情境下購買產品。
  8. 過去的官網發展是由各產品部門，比照他們對產品的銷售瞭解去規劃的，是由不同產品部門意見彙整成的，ASUS希望我們多研究官網為何這麼做。
  9. 目前官網的筆電分類方式是分為，輕薄類、效能類.... 等。原因是ASUS的產品線有幾百條，希望透過這方式讓消費者找到最適合他們的產品。
  10. ASUS的產品線相當多，但有90%的產品是消費者不瞭解的，只有10%主力產品有媒體線推廣。
  11. 希望提高購買前的比例>支援服務
  12. ASUS希望改善目前網頁的流程體驗，新功能發展是其次。
三、GA問題請教
  • All web site Data 是全部網站的數據
  • official 為官網數據
  • shop.asus.com 為福利品網頁數據
  • store.asus 為EC數據
四、TA比例請教
  • 男性大於女性
  • 不同產品線的TA數據不同
  • TA並非他們的主線，產品銷售才是主要目標
五、ASUS分享Trello

　１. Persona： 28歲Lisa，想買筆電但預算不高，購買方式為論壇搜尋、官網或實體通路購買。
　　  Touch Point：
  • 在官網想瞭解目前市價，但產品頁面裡沒有訂價資訊。
  • 官網內無Lisa要的UX305筆電型號。
  • 手機板網頁無法列比較清單。
  • 在官網產品頁面點選立即購買後，卻跳至福利品頁面。
  • 福利品phone版網頁無響應式設計。
  • 使用手機購買的過程中，點選下次購買，卻跳至結帳頁面。
  • 購買的過程中需登錄會員(無快速登入服務)，當他填完所有資料後，網頁卡住，重新整理後發現資料都不見了很困擾。
  • 瞭解訂單狀況時，從google搜尋ASUS訂單狀況，卻跳至shop且需選擇語言。
  • 在官網、EC尋找產品時，有許多照片太相像，易混淆使用者。
  • 筆電並非常購買的東西，卻被要求要註冊會員(以使用者角度來看會覺得沒有好處)
  • 會員中心沒有查詢訂單功能，且系統信也沒有明確的資訊可查詢。
  • 網站上只提供0800，卻無法用手機撥打，必須要用市話才能撥打。

　2.  Persona：27歲上班族，網管人員，愛好科技產品。
　　  Touch Point：
  • 官網上的分類不清楚，使用者找不到顯示卡的細項分類。
  • 網站上的驅動程式是一年前的舊版本。
  • 線上說明書字體太小。
  • 搜尋常見問題＂組件產生畢畢聲＂卻得不到結果。(但論壇上有諸多類似問題)

　3.  Persona：33歲上班族，網管人員，愛好科技產品。
　　  Touch Point：
  • 技術規格重覆的資訊很多，無法快速比較出不同品牌間的差異。
  • 照片解析度不夠，無法詳細閱讀規格。
  • 使用者沒有主動註冊習慣，遇到問題是去論壇而非官網。

DITL finding: 
1) Main site 設計需考量3種族群：
    1. 有目性的查找 （被廣告吸引，或品牌知名度高的國家-台灣區）
    2. 隨意瀏覽型？ （知名度與行銷較低的其他國家）
     需解決：商品查找，與產品分類＆如何讓眾多產品線容易/清楚地被找
2) 網站設計的結果恐由各部門角力拉鋸而來，真正的使用者需求反而被弱化了（這不是ditl的解決責任，但需要了解背景）
3) TA 要側重的族群：end-user v.s 技術人員 （需要與對方再確認）
4) 公司除了GA 同事和PM？, 大家對數據的概念是模糊的, 下週CRM同事會分享相關數據
5) ec 裏的store 和福利品是交由廠商設計; 網站中的其他子站，也多是其他部門自行管理 （因此凸顯了guide line的重要性，與實際落地的難度...）
6) 會員中心的資料，將作為big data計劃的主要來源，協助未來產品銷售的預測，並針對會員偏好做為產品設計的方向
