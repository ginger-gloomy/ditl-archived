請見附檔為會議中討論的簡報, 因為裡面有數據資料屬於公司機密, 請內部參考就好不要外流或做其他使用.

會議時間, 方便的話請安排在11/1 (二) 早上10:00在台科.

另關於此次後測的涵蓋範圍與任務安排, 我們的想法如下:
l 以上次的五個主要任務為主, 涵蓋購買前, 購買中, 購買後, 但其中幾個子任務抽換成新的 à 那些要抽換, 我們可以在會議上討論
l 主要任務
n 購買前(找產品)
Ø 找產品方式: 指定型號, 不指定型號(依使用者需求來找)
Ø 需涵蓋範圍: 完整瀏覽路徑(L1àL2àL3à PD), Banner輪播, 頭帶式產品分類, Help Me Choose, Compare, Spec Filter, 瀑布式設計
n 購買中
Ø 僅走到PD頁的Buy Now即可, 或是Where to buy頁面, Store的購買流程不測
n 購買後(解決問題)
Ø 以測試者購買的產品發想, 發生產品使用問題, 引導他至官網相關服務找尋問題解答
Ø 分為三條路: FAQ, Contact us, ICR
n Support Site vs ICR
Ø 以幫助/解決測試者的角度出發, 設計同樣的問題+任務, 於Support Site與ICR上測試 à 主要用意是看ICR是否有辦法完全取代Support Site
Ø 測試子任務: FAQ, Contact us, Driver download, 找尋維修據點
l 測試產品線
n 以瀏覽量最高的產品線為主, 可於任務中穿插使用, 不用每條產品線都測全部任務
Ø Phone, Notebook, Motherboard, Networking
l 後測目標, 預期效果
n 驗證官網V3.0版的改版方向對於使用者來說是有幫助的
n Support site相關的測試結果, 使用者回饋可作為後續改版參考
n PC, 手機版不同設計對於使用者的瀏覽體驗有幫助

也請台科同學們協助構思一下任務規劃, 下次會議中再討論細節囉.

謝謝.
